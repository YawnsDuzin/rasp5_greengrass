## 1. AWS 계정 생성

### 1.1 AWS 홈페이지 접속

1. 웹 브라우저에서 **https://aws.amazon.com** 접속
2. 우측 상단의 **"AWS 계정 생성"** 또는 **"Create an AWS Account"** 클릭

### 1.2 계정 정보 입력

```
┌─────────────────────────────────────────────────────────────┐
│                    AWS 계정 생성                             │
├─────────────────────────────────────────────────────────────┤
│  이메일 주소: [your-email@example.com        ]              │
│                                                             │
│  AWS 계정 이름: [MyAWSAccount                 ]             │
│  (나중에 변경 가능, 영문으로 입력 권장)                        │
│                                                             │
│  [계속]                                                      │
└─────────────────────────────────────────────────────────────┘
```

- **이메일 주소**: 자주 사용하는 이메일 (중요 알림 수신용)
- **AWS 계정 이름**: 본인이 식별할 수 있는 이름 (예: "MyIoTProject")

### 1.3 이메일 인증

입력한 이메일로 인증 코드가 전송됩니다. 메일함을 확인하여 6자리 코드를 입력하세요.

### 1.4 루트 사용자 비밀번호 설정

```
비밀번호 요구사항:
✓ 최소 8자 이상
✓ 대문자, 소문자, 숫자 포함
✓ 특수문자 포함 권장 (!@#$%^&*)
```

> ⚠️ **중요**: 이 비밀번호는 AWS의 모든 권한을 가진 "루트 사용자" 비밀번호입니다.
> 안전한 곳에 보관하고, 일상적인 작업에는 IAM 사용자를 만들어 사용하세요.

### 1.5 연락처 정보

```
┌─────────────────────────────────────────────────────────────┐
│  계정 유형: (●) 개인  ( ) 비즈니스                           │
│                                                             │
│  이름: [홍길동                              ]                │
│  전화번호: [010-1234-5678                  ]                │
│  국가/리전: [대한민국                       ]                │
│  주소: [서울시 강남구...                    ]                │
│  시/도: [서울                              ]                 │
│  우편번호: [06000                          ]                 │
└─────────────────────────────────────────────────────────────┘
```

### 1.6 결제 정보

```
┌─────────────────────────────────────────────────────────────┐
│                      결제 정보                               │
├─────────────────────────────────────────────────────────────┤
│  신용카드/체크카드 번호: [                    ]              │
│  만료일: [MM/YY]                                            │
│  카드 소유자 이름: [                          ]              │
│  CVV: [   ]                                                 │
└─────────────────────────────────────────────────────────────┘
```

> **참고**: $1 정도의 인증 결제가 발생할 수 있으며, 이는 즉시 취소됩니다.

### 1.7 본인 인증

전화번호로 전송된 코드를 입력하여 본인 확인을 완료합니다.

### 1.8 지원 플랜 선택

```
┌─────────────────────────────────────────────────────────────┐
│                     지원 플랜 선택                           │
├─────────────────────────────────────────────────────────────┤
│  (●) Basic Support - 무료                                   │
│      · AWS 문서 및 포럼 접근                                 │
│      · 서비스 상태 대시보드                                   │
│      · 청구 및 계정 지원                                     │
│                                                             │
│  ( ) Developer - $29/월                                     │
│  ( ) Business - $100/월~                                    │
│  ( ) Enterprise - 맞춤형                                     │
└─────────────────────────────────────────────────────────────┘
```

**"Basic Support - 무료"**를 선택합니다. 개인 프로젝트에는 충분합니다.

### 1.9 계정 생성 완료

축하합니다! AWS 계정이 생성되었습니다.
"AWS Management Console로 이동" 버튼을 클릭합니다.

```
│   │  · EC2                    ☆ 서비스를 추가하세요              │    │
```

### 2.3 주요 메뉴 설명

| 위치 | 기능 |
|------|------|
| **서비스** | 모든 AWS 서비스 목록 |
| **검색창** | 서비스나 기능 빠른 검색 |
| **리전** | 사용할 데이터센터 위치 선택 |
| **계정** | 계정 설정, 보안 자격 증명, 로그아웃 |

검색창에 다음 서비스들을 검색해서 위치를 확인해두세요:

- **IoT Core** - IoT 디바이스 관리
- **IoT Greengrass** - 엣지 컴퓨팅 서비스
- **IAM** - 사용자 및 권한 관리
- **S3** - 파일 저장소
- **CloudWatch** - 모니터링 및 로그

## 3. 리전 선택

### 3.1 리전이란?

AWS는 전 세계에 여러 **리전(Region)**이라는 데이터센터 그룹을 운영합니다.
서비스 속도와 비용을 위해 가까운 리전을 선택하는 것이 좋습니다.

### 3.2 한국에서 권장 리전

| 리전 코드 | 리전 이름 | 위치 | 권장도 |
|-----------|----------|------|--------|
| `ap-northeast-2` | 아시아 태평양 (서울) | 한국 | ⭐⭐⭐ 최우선 |
| `ap-northeast-1` | 아시아 태평양 (도쿄) | 일본 | ⭐⭐ 차선 |
| `ap-southeast-1` | 아시아 태평양 (싱가포르) | 싱가포르 | ⭐ |

### 3.3 리전 변경 방법

```
우측 상단의 리전 선택 드롭다운 클릭
         │
         ▼
┌─────────────────────────────┐
│ 리전 선택                    │
├─────────────────────────────┤
│ ● 아시아 태평양 (서울)       │  ← 선택
│   ap-northeast-2            │
│ ○ 아시아 태평양 (도쿄)       │
│   ap-northeast-1            │
│ ○ 미국 동부 (버지니아 북부)  │
│   us-east-1                 │
│ ...                         │
└─────────────────────────────┘
```

> ⚠️ **중요**: 리전마다 생성한 리소스가 별도로 관리됩니다.
> 항상 같은 리전(**서울: ap-northeast-2**)에서 작업하세요!

### 3.4 리전 확인

콘솔에서 작업 전 항상 우측 상단에서 **"서울"** 또는 **"ap-northeast-2"**가
선택되어 있는지 확인하세요.

## 4. 결제 알림 설정

예상치 못한 비용 발생을 방지하기 위해 결제 알림을 설정합니다.

### 4.1 Billing 대시보드 접속

1. 우측 상단 계정 이름 클릭
2. **"결제 대시보드"** 또는 **"Billing Dashboard"** 클릭

### 4.2 예산 알림 생성

1. 좌측 메뉴에서 **"예산(Budgets)"** 클릭
2. **"예산 생성(Create budget)"** 클릭

```
┌─────────────────────────────────────────────────────────────┐
│                      예산 생성                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  예산 유형 선택:                                             │
│  (●) 비용 예산 - 템플릿 사용 (간편)                          │
│  ( ) 사용량 예산                                            │
│  ( ) 절감형 예산                                             │
│                                                             │
│  템플릿:                                                     │
│  (●) 월별 비용 예산                                          │
│                                                             │
│  예산 금액: [$5                    ]                        │
│                                                             │
│  이메일 수신자: [your-email@example.com    ]                │
│                                                             │
│  [예산 생성]                                                 │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 알림 임계값

기본적으로 다음 경우에 알림이 발송됩니다:
- 예산의 85% 도달 시
- 예산의 100% 도달 시
- 예산 초과 예상 시

## 5. AWS CLI 설치

AWS CLI(Command Line Interface)는 터미널에서 AWS 서비스를 제어하는 도구입니다.
라즈베리파이에서 Greengrass 설정 시 필요합니다.

### 5.1 개발 PC에 AWS CLI 설치

#### Windows

```powershell
# PowerShell에서 실행
msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2.msi
```

또는 https://awscli.amazonaws.com/AWSCLIV2.msi 다운로드 후 설치

#### macOS

```bash
# Homebrew 사용
brew install awscli

# 또는 공식 패키지
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /
```

#### Linux (Ubuntu/Debian)

```bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
```

### 5.2 설치 확인

```bash
aws --version
# 출력 예: aws-cli/2.15.0 Python/3.11.6 Linux/5.15.0-91-generic exe/x86_64.ubuntu.22
```

### 5.3 AWS CLI 자격 증명 설정

> ⚠️ 이 단계는 **02. IAM 설정** 문서에서 IAM 사용자를 만든 후 진행합니다.

```bash
aws configure
```

```

Default output format [None]: json
```
